FTP / SCP / rsync — Transfer Cheat Sheet

sftp scp rsync
FTP / SFTP File Transfer Protocol
Connection Open a session ▶
ftp <host> # plain FTP — cleartext, avoid in production sftp user@host # secure FTP over SSH sftp -P 2222 user@host # non-default port sftp -i ~/.ssh/id_rsa user@host # specify identity file
Upload Interactive — push files to remote ▶
put /local/file.tar /remote/ # upload single file mput *.tar.gz # upload multiple files (glob) put -r /local/dir /remote/dir # upload directory (sftp only)
Download Interactive — pull files from remote ▶
get /remote/file.tar /local/ # download single file mget *.log # download multiple files (glob) get -r /remote/dir /local/dir # download directory (sftp only)
Navigation Session navigation & misc ▶
ls / lls # list remote / local directory cd /remote/path / lcd /local # change remote / local directory pwd / lpwd # print remote / local working dir bye / quit # close session
Batch Non-interactive / scripted transfers ▶
# Here-doc batch transfer in shell script sftp user@host <<EOF put /backup/db.tar.gz /drop/ bye EOF # Execute commands from a pre-written batch file sftp -b /tmp/batch.txt user@host

Note: Plain FTP sends credentials in cleartext — use SFTP or SCP in production. Disable anonymous FTP on backup servers.

SCP Secure Copy Protocol
Local → Remote Push files to a remote host ▶
scp file.tar user@host:/backup/ # copy single file to remote scp -r /local/dir user@host:/backup/ # copy directory recursively scp -P 2222 file.tar user@host:/tmp/ # non-default SSH port scp -i ~/.ssh/id_rsa file.tar user@host:/tmp/ # specify identity file scp -C file.tar user@host:/backup/ # enable compression in transit scp -l 8192 big.tar user@host:/bkp/ # limit bandwidth (Kbit/s)
Remote → Local Pull files from a remote host ▶
scp user@host:/backup/file.tar /local/ # pull single file from remote scp -r user@host:/backup/dir /local/ # pull directory from remote
Remote → Remote Copy between two remote hosts ▶
scp user@src:/data/file.tar user@dst:/data/
Useful Combos Flags worth knowing ▶
scp -rp /local/dir user@host:/dst/ # preserve timestamps & permissions scp -q file.tar user@host:/bkp/ # quiet mode — suppress progress scp -v file.tar user@host:/bkp/ # verbose — useful for debugging

Note: SCP lacks resume capability. For large transfers or unreliable links use rsync over SSH instead. On OpenSSH >= 9.0, scp uses the SFTP protocol internally.

rsync Remote Sync
Core Flags Key options reference ▶
-a (--archive) # recursive + preserve perms, times, symlinks, owner, group -z (--compress) # compress data in transit -v (--verbose) # show files being transferred -P # --partial + --progress (resume + show progress) -n (--dry-run) # simulate — no changes written --delete # delete files on dst not present on src --checksum # compare by checksum, not mtime/size --bwlimit=5000 # limit bandwidth (KBytes/s)
Local → Remote Push over SSH ▶
rsync -avz /src/ user@host:/dst/ # sync dir (trailing / = contents only) rsync -avzP /src/ user@host:/dst/ # with progress & resume rsync -avz --delete /src/ user@host:/dst/ # mirror — removes files deleted on src rsync -avz -e "ssh -p 2222" /src/ user@host:/dst/ # non-default SSH port rsync -avz -e "ssh -i /root/.ssh/id_rsa" /src/ user@host:/dst/ # custom identity

Trailing slash matters: /src/ syncs contents into /dst/ — /src (no slash) creates /dst/src/ inside the destination.

Remote → Local Pull from remote host ▶
rsync -avz user@host:/src/ /local/dst/ # pull remote directory locally rsync -avzP user@host:/src/big.tar /local/ # pull single file with resume
Local → Local Local disk sync ▶
rsync -av /mnt/data/ /mnt/backup/ # local disk sync
Filtering Include / exclude patterns ▶
rsync -av --exclude='*.log' /src/ user@host:/dst/ # exclude by pattern rsync -av --exclude-from=/tmp/excl.txt /src/ user@host:/dst/ # exclude list from file rsync -av --include='*.tar.gz' --exclude='*' /src/ user@host:/dst/ # include only .tar.gz
Backup / AIX Production & AIX-specific considerations ▶
rsync -avz --numeric-ids /src/ user@host:/dst/ # preserve UID/GID numerically (cross-host) rsync -avz --log-file=/var/log/rsync.log /src/ user@host:/dst/ # write transfer log rsync -avzn /src/ user@host:/dst/ 2>&1 | tee /tmp/dryrun.out # dry-run with captured output

AIX: rsync is not native — install via IBM Linux Toolbox or Bull Freeware RPM. Always use SSH key auth for scripted/cron jobs. Use --numeric-ids when syncing between AIX and Linux to avoid UID/GID mapping mismatches.

⚠ Key Notes
  • Plain FTP sends credentials in cleartext — always use SFTP or SCP in production environments.
  • SCP has no resume capability — for large backup archives over unreliable links, use rsync -P instead.
  • Trailing slash on rsync source is critical: /src/ syncs contents into dst; /src (no slash) creates a subdirectory inside dst.
  • Always dry-run (-n) before executing any rsync job with --delete — there is no undo.
  • rsync --bwlimit is in KBytes/s; scp -l is in Kbit/s — they are not the same unit.
  • On AIX, rsync requires installation via toolbox/rpm and SSH key auth must be configured for any scripted job.
  • Use --numeric-ids when syncing between AIX and Linux to prevent UID/GID remapping on the destination.
